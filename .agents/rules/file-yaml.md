# YAML Formatting Expectations

**Context**: These rules apply to all YAML and YML files across the repository to ensure consistent formatting and readability.

## Rules

- Use 2 spaces for indentation. Do not use tabs.
- Ensure consistent spacing after colons in key-value pairs (e.g., `key: value`).
- Start lists with a hyphen followed by a single space.
- End files with a single newline character.
- Avoid trailing whitespace on lines.
- Keep YAML documents well-structured and use standard YAML conventions.
